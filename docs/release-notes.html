商店上架版本更新紀錄

給 Chrome Web Store 與 Firefox Add-ons 使用者的更新摘要,只列出對使用者直接有感的功能變更與重要修正。完整的版本歷史(含每一個 patch 與技術細節)請見 GitHub CHANGELOG.md。

v1.8.67 2026-05-09 上架

相對於上一個商店版本 v1.8.55

這版的兩大主軸：翻譯目標與介面雙雙擴充為 8 種語言、複雜頁面雙語對照模式大幅收斂。

  • 多語翻譯目標：翻譯目標從繁體中文擴充為 8 種語言（繁體中文、簡體中文、英文、日文、韓文、西班牙文、法文、德文）。在設定頁的「翻譯目標語言」分頁切換，所有翻譯模式（網頁、PDF、YouTube 字幕、自訂模型）共用此設定；預設依瀏覽器語言自動推導。
  • 多語介面：Popup、設定頁、網頁翻譯進度通知、PDF 文件翻譯閱讀器全部 i18n 化，同樣支援 8 種語言。介面語系可以跟翻譯目標語言獨立設定（例如「英文介面 + 翻成繁中」）。
  • CJK 譯文字形變體正確：翻成日文 / 韓文時，譯文採用對應 locale 的字體 stack，避免站點原本鎖死繁中字體時譯文出現字形錯位（例如日文「査」字身仍是中文版）。
  • GitHub / Medium 等複雜頁面雙語對照大幅修補：雙語對照模式累積 15+ 條結構性修補，例如 GitHub 程式碼區塊不再被翻譯蓋掉、檔名欄位不亂翻、表格 cell 跟譯文之間有像樣呼吸空間、commit 日期不再觸發 LLM 幻想長文、Medium 引用文字譯文不衝出 column、按讚 / 留言計數不被誤翻等。
  • 自訂 Provider 強化段序號標記（預設開啟）：自訂 OpenAI 相容 Provider 連本機量化模型（Ollama / llama.cpp 跑 gemma-4 / Qwen 量化版等）時，改用更穩固的段序號標記，避免被誤翻成「N1、N2」等殘留洩漏到譯文。商用 API 使用者（OpenRouter / DeepSeek / Groq 等）可在「自訂模型」分頁關掉省 token。
  • 隱私政策完整更新 + 英文版：隱私政策補上 8 語 target、3 引擎（Gemini / Google Translate / OpenAI 相容）、5 條對外連線（含匯率 API / GitHub release check）的最新狀態，並新增英文版。
  • Log 系統修補：設定頁 Log 分頁打開後能看到背景 service worker 重啟前的歷史紀錄、清除按鈕真的清乾淨（原本 SW 重啟後舊紀錄又會冒出來）。

v1.8.55 2026-05-05 上架

相對於上一個商店版本 v1.8.38

這版以 PDF 文件翻譯與雙語體驗為主軸。

  • PDF 文件翻譯(beta):把 PDF 拖到「翻譯文件」分頁,Shinkansen 會在本機解析版面、送翻譯,並以左右並排的雙頁對照模式呈現,供使用者閱讀。可下載成雙語對照 PDF。整份文件全程在瀏覽器內處理,不會上傳到任何伺服器,只有抽取出來的純文字會送到使用者自己設定的翻譯端點。
  • PDF 譯文編輯:閱讀器內可逐段編輯譯文,支援基本的 markdown 格式(粗體、斜體、連結),內建找/取代功能。儲存後重新生成雙頁預覽。
  • 文章術語表:翻譯前可請 AI 從文章內抽取人名、專有名詞對照表,確認或編輯後再翻譯,確保整份文件的譯名前後一致。對照表可匯入或匯出 JSON。
  • Google Drive 影片字幕翻譯:YouTube 字幕翻譯機制延伸到 Drive 上的影片,字幕樣式與雙語對照行為一致。
  • YouTube 雙語字幕視覺整合:雙語模式下中英文字幕共用一塊黑底,徹底解決原本「兩行英文擠掉譯文」與「視覺三塊黑底污染」的問題。
  • 雙語對照譯文強調色自訂:雙語對照模式提供 8 種強調色與深色模式自動配色,深色頁面下對比加強。
  • 翻譯記憶不再隨版本更新清除:擴充功能版本變更時不再自動清除翻譯快取,避免使用者每次升級都掉資料。如需清除,可在 popup 或設定頁手動清。
  • 顯示貨幣切換(限繁中介面):設定頁費用統計可在 USD 與 TWD 之間切換,自動換算。

v1.8.38 2026-05-01 上架

相對於上一個商店版本 v1.8.30

這版以 UX 與跨瀏覽器相容性修補為主。

  • 本機 Ollama / llama.cpp 連線解法說明:設定頁的自訂 Provider 分頁加入「連本機 Ollama / llama.cpp 收到 403?」展開區,內含可行的兩種解法(Ollama 設定 OLLAMA_ORIGINS、或用 hosts 檔指 fake domain)。
  • Arc 等不支援 openOptionsPage 的瀏覽器支援:在 Arc 上點 popup 的「設定」按鈕現在會正常開啟設定頁。
  • YouTube 關閉 CC 後不再持續送翻譯請求:使用者按 YouTube 字幕按鈕關閉 CC 時,Shinkansen 會立即暫停字幕翻譯,並隱藏畫面上殘留的中文字幕,避免在使用者已關掉字幕的情況下繼續耗用 token。
  • 雙語對照在深色背景頁面對比改善:雙語對照 wrapper 現在會自動偵測頁面實際背景亮度,深色頁面採用對比加強的配色,避免原本米色底跟父層淺灰文字對比破裂。同時加入圓角、padding 微調、與下一段間距處理。
  • 論壇 / 訂閱中 thread 標題偵測修復:vBulletin 等論壇上「已訂閱中」的 thread(會多包一層 wrapper 與小圖示)標題現在能正確被翻譯。

v1.8.30 2026-04-30 上架

相對於上一個商店版本 v1.4.22

這版距離 v1.4.22 累積了大量功能。以下是對使用者影響最大的更新。

  • 極速秒翻:按下翻譯後 1 秒就看到頁面開始變中文,不必等整批 API 回完才一次塞回頁面(限 Gemini 引擎)。文章前段段落改走 streaming,首字延遲從 2.5–4.4 秒大幅縮短到 1.0–1.2 秒。
  • 雙語對照模式:popup 一鍵切換顯示模式。「單語覆蓋」原地替換、「雙語對照」原文保留,譯文以新段落附加。雙語模式提供 4 種視覺標記(淡底色、左邊細條、波浪底線、無)讓使用者選擇譯文呈現風格。
  • 自訂 AI 模型:除了既有的 Google Gemini 與 Google Translate,新增「自訂 OpenAI 相容端點」分頁,可接 OpenRouter、Together、DeepSeek、Groq、Fireworks、Anthropic Claude、本機 Ollama / llama.cpp 等任何 OpenAI 相容服務,使用百種以上模型。
  • YouTube 自動字幕智慧分句:YouTube 自動產生的字幕(沒有人工字幕的影片,CC 標記為 auto-generated)原本是按時間切割,每條只有 1–3 個英文字、沒有標點,逐條翻譯失去語意上下文。現在會由 AI 重新依語意分句、補上標點再翻譯,中文字幕從「破碎的詞」變「完整的句子」。預設走「混合模式」,先用本地分句快速顯示,背景同時跑 AI 分句,完成後用更精緻版本替換。
  • 只翻文章開頭(節省費用):翻譯只跑頁面前幾段(範圍 5-50,預設 25 段),大幅節省 token 用量,適合先預覽再決定是否要看完整篇。翻譯完成後右下角會出現「翻譯剩餘段落」按鈕,點按可繼續翻完整篇,前面已翻好的段落會從本地快取載入,不重複收費。
  • 禁用詞清單:可由使用者編輯的禁用詞對照表(預設 25 條,例如「視頻、軟件、數據、網絡、用戶」),明確要求 AI 不要使用左欄詞彙。
  • 翻譯快速鍵與三組預設:三組可自訂的翻譯預設,每組各綁一個快速鍵(macOS Option+A/S/D、Windows Alt+A/S/D),引擎、模型、標籤都可在設定頁「翻譯快速鍵」自訂。已翻譯狀態下按任一快速鍵還原,翻譯中按任一快速鍵取消。
  • 跨 tab 延續翻譯:在分頁 A 翻譯後,從 A 點連結開新分頁 B(含 Cmd / Ctrl+點連結、target="_blank"、JavaScript 開啟),分頁 B 會自動翻譯並繼承同一組預設。多層延續(B 開 C 也會繼續)。
  • 翻譯啟動更快:長網頁(Wikipedia、論壇、長 Medium)翻譯啟動明顯變快。長頁的首字延遲在多個典型網站平均減少 29%(NPR 從 11.7 秒降到 5.1 秒)。
  • 看到譯文順序更直覺:長網頁翻譯時最先看到的譯文從「導覽列、cookie 同意書、目錄」改為「文章標題與第一段內文」(先翻 main / article 內的段落)。
  • 商店升級提示:Chrome Web Store / Firefox Add-ons 自動更新到新版時,popup 與 toast 會顯示「歡迎升級」提示,列出本版重點功能。Patch 級的小更新不會打擾。
  • 「翻譯本頁」工具列按鈕:工具列圖示現在可以直接觸發翻譯,並可在設定頁指定使用哪一組預設。
  • 背景分頁 CPU 與電量降耗:當分頁切到背景時,Shinkansen 會暫停內容偵測與 SPA 網址輪詢,減少背景分頁的耗電。
  • 用量紀錄可即時重新載入:設定頁的用量紀錄分頁加入「重新載入」按鈕,不必關閉設定頁就能看到最新紀錄。
  • 多項 YouTube 字幕體驗改善:字幕完全旁路 YouTube 原生「一字一字跳」的問題、字幕樣式自動對齊原生 YouTube 字幕、字幕分頁重整(自動翻譯、引擎、進階、視窗、Prompt)、字幕也可套用固定術語表與禁用詞清單(預設關)。
